Unit 2 · Data Structures & Algorithms
~25% · High Priority ▾

📚 Stack (LIFO)

  • Push → append(), Pop → pop(), Peek → stack[-1]
  • isEmpty() → len(stack)==0
  • Applications: Undo/Redo, Browser Back, Balanced Brackets, Infix→Postfix
  • Time: O(1) for push/pop

🚶 Queue (FIFO)

  • Enqueue → append() rear, Dequeue → pop(0) front
  • Better: collections.deque se O(1) dequeue
  • Circular Queue, Priority Queue, Deque
  • Applications: CPU Scheduling, BFS, Print Spooler

🔗 Linked List

  • Node = data + next pointer
  • Singly → Doubly → Circular LL
  • No random access (O(n) traversal)
  • Dynamic size — array se better for insert/delete at beginning

🌳 Trees

  • Binary Tree: max 2 children per node
  • BST: Left < Root < Right
  • Inorder=LNR (sorted!), Preorder=NLR, Postorder=LRN
  • Level Order=BFS, Height = longest root-to-leaf path
  • n nodes wale BST mein max height = n-1

🔍 Searching

  • Linear Search: O(n), works on unsorted array
  • Binary Search: O(log n), SORTED array only
  • Binary Search: mid = (low+high)//2
  • Number of comparisons = log₂(n)+1 (worst case)

🔃 Sorting

  • Bubble: adjacent swap, O(n²), stable
  • Selection: min select karo, O(n²), not stable
  • Insertion: sorted part mein insert, O(n²), stable
  • Merge Sort: O(n log n) always, O(n) space, stable
  • Quick Sort: avg O(n log n), worst O(n²) pivot pe depend
✅
Golden Rule: BST ka Inorder Traversal hamesha ascending sorted order deta hai. Ye CUET mein guarantee aata hai! Also: Stable sort = Bubble, Insertion, Merge. Unstable = Selection, Quick.

Unit 3 · DBMS & SQL
~20% ▾

📐 Concepts & Keys

  • Primary Key: Unique + NOT NULL, ek table mein ek hi PK
  • Foreign Key: Dusre table ka PK — referential integrity
  • Candidate Key: Sab possible unique identifiers
  • Super Key: Candidate key + extra attributes
  • Degree = columns | Cardinality = rows

🔎 SQL Query Types

  • DDL: CREATE, ALTER, DROP, TRUNCATE
  • DML: INSERT, UPDATE, DELETE, SELECT
  • DCL: GRANT, REVOKE
  • TCL: COMMIT, ROLLBACK, SAVEPOINT
  • DISTINCT, ORDER BY ASC/DESC

🔢 Aggregate Functions

  • COUNT(*), SUM(col), AVG(col), MAX(col), MIN(col)
  • GROUP BY col HAVING condition
  • WHERE: rows filter (GROUP BY se PEHLE)
  • HAVING: groups filter (GROUP BY ke BAAD)
  • Aggregate functions WHERE mein nahi chalte!

🔀 Joins

  • INNER JOIN: Sirf matching rows dono tables se
  • LEFT JOIN: Sab left + matched right (NULL if no match)
  • RIGHT JOIN: Sab right + matched left
  • CROSS JOIN: Cartesian product (m × n rows)
  • Subquery: WHERE ke andar SELECT
💡
Key difference yaad karo: WHERE individual rows filter karta hai, HAVING groups filter karta hai. Aggregate functions sirf HAVING mein allowed hain, WHERE mein nahi.
SQL-- CUET favourite query pattern SELECT Department, COUNT(*) AS Total, AVG(Salary) AS AvgSal FROM Employee WHERE Salary > 30000 GROUP BY Department HAVING COUNT(*) >= 3 ORDER BY AvgSal DESC;

Unit 4 · Computer Networks
~15% ▾

📦 OSI Model — 7 Layers

  • Trick: All People Seem To Need Data Processing
  • 7. Application: HTTP(80), HTTPS(443), FTP(21), SMTP(25), DNS
  • 6. Presentation: Encryption, Compression, Format
  • 5. Session: Session establish/manage/terminate
  • 4. Transport: TCP (reliable), UDP (fast), Port numbers
  • 3. Network: IP, Routing — Router is here (Layer 3)
  • 2. Data Link: MAC Address, Frames — Switch is here (Layer 2)
  • 1. Physical: Bits, Cables — Hub/Repeater here (Layer 1)

🌍 IP Addressing

  • IPv4: 32-bit, 4 octets (e.g. 192.168.1.1)
  • IPv6: 128-bit, hexadecimal — address exhaustion solve
  • Class A: 0-127 | B: 128-191 | C: 192-223
  • Private: 10.x.x.x, 172.16-31.x.x, 192.168.x.x
  • DHCP: Auto IP assign | DNS: Domain → IP translate
  • TCP/IP Model = 4 layers (vs OSI = 7 layers)

🔌 Network Devices

  • Hub (L1): Broadcast to all — no intelligence, obsolete
  • Switch (L2): MAC table — smart selective forwarding
  • Router (L3): IP routing between different networks
  • Bridge (L2): 2 LAN segments connect karta hai
  • Repeater (L1): Signal regenerate/amplify karta hai
  • Gateway: Protocol translation (all layers)

🔐 Cybersecurity

  • Virus: Self-replicates, files infect karta hai
  • Worm: Network se spread, no host file needed
  • Trojan: Useful dikhta hai but harmful
  • Ransomware: Data encrypt kare, ransom maange
  • Phishing: Fake site se credentials steal
  • DoS/DDoS: Server overwhelm karna
  • SSL/TLS: Encrypted communication
🚨
TCP vs UDP (CUET mein aata hai): TCP = connection-oriented, reliable, ordered, slower (HTTP, email, FTP). UDP = connectionless, fast, no guarantee (streaming, DNS, gaming, VoIP).

Unit 5 · Operating System Concepts
~4% ▾

🖥️ Process Management

  • Process states: New → Ready → Running → Waiting → Terminated
  • PCB: Process Control Block — process info store karta hai
  • Thread = lightweight process (shared memory)
  • Context switching = CPU processes ke beech switch karna

🔄 CPU Scheduling

  • FCFS: First Come First Serve — simple, convoy effect
  • SJF: Shortest Job First — min avg waiting time
  • Round Robin: Fixed time quantum — fair, preemptive
  • Priority: Higher priority first — starvation possible

🧠 Memory Management

  • Paging: Fixed-size pages, no external fragmentation
  • Segmentation: Variable-size logical segments
  • Virtual Memory: RAM + Disk (swap space)
  • Thrashing: Excessive page faults — performance crashes

📁 File System

  • Allocation: Contiguous, Linked, Indexed
  • Deadlock conditions (sab 4 ek saath): Mutual Exclusion, Hold & Wait, No Preemption, Circular Wait
  • Cache: L1 fastest/smallest, L3 slowest/largest

  • Sec 66: Hacking / computer-related offences
  • Sec 67: Obscene material online publish karna
  • Sec 72: Privacy breach / confidentiality

🔒 IPR & Open Source

  • Patent: Inventions — 20 years protection
  • Trademark: Brand names / logos
  • Open Source (FOSS): GPL, MIT, Apache licenses
  • Creative Commons: 6 license types
  • Plagiarism: Others ka kaam apna batana

🤖 Emerging Technologies

  • AI Types: Narrow AI, General AI, Super AI
  • ML: Supervised, Unsupervised, Reinforcement
  • IoT: Smart devices interconnected via internet
  • Cloud: SaaS (apps), PaaS (platform), IaaS (infrastructure)

🌱 Digital Safety

  • Netiquette: Online behaviour rules
  • E-waste: Electronic waste — toxic materials

PYQ Trends & Sample Questions (2022–2025)
MUST SOLVE ▾
🗓 CUET CS 2022
Q: Python mein output kya hoga? x = [1,2,3,4,5]; print(x[1:4:2])
✅ Answer: [2, 4] — Index 1 se 3 tak, step 2 → positions 1,3 → values 2 aur 4
🗓 CUET CS 2022
Q: Primary Key ke baare mein kaunsa statement GALAT hai? (a) Ek se zyada PK ho sakti hain (b) NULL nahi hoti (c) Unique hoti hai (d) FK reference kar sakti hai
✅ Answer: (a) — Ek table mein sirf EK Primary Key hoti hai (composite bhi ek counted hoti hai)
🗓 CUET CS 2023
Q: Stack operations: Push(10), Push(20), Pop(), Push(30), Pop() ke baad stack mein kya hoga?
✅ Trace: [10] → [10,20] → pop 20 → [10] → [10,30] → pop 30 → [10]. Answer: [10], Top = 10
🗓 CUET CS 2023
Q: BST Inorder traversal: Root=8, Left subtree={3,1,6,4,7}, Right subtree={10,14,13}. Output?
✅ Answer: 1,3,4,6,7,8,10,13,14 — BST Inorder hamesha ascending sorted order deta hai
🗓 CUET CS 2024
Q: SELECT COUNT(*) FROM Student WHERE Marks > 75 GROUP BY Class HAVING COUNT(*) >= 3 — Kya return karega?
✅ Answer: Har Class ka count jahan Marks>75 wale students 3 ya zyada hain. WHERE pehle, phir GROUP BY, phir HAVING.
🗓 CUET CS 2025
Q: Hub aur Switch mein kya fark hai? Hub kis OSI layer pe kaam karta hai?
✅ Hub → Physical Layer (L1) → Broadcast to ALL ports. Switch → Data Link Layer (L2) → MAC table use karta hai, specific port pe send karta hai.
